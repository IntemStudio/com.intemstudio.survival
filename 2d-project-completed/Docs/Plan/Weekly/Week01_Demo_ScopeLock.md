## Goal

1주차의 목표는 데모 개발을 시작하기 전에 **데모에 넣을 것과 뺄 것**을 확정하는 것이다. 이번 주에는 큰 기능 구현보다 결정, 기준선 테스트, 작업 목록 축소를 우선한다.

주가 끝날 때는 `Plan_Release_Roadmap.md`의 `Demo Must`가 현실적인 크기로 정리되어 있어야 한다. 이후 2주차부터는 이 결정에 맞춰 무기 성장, 보스/특수몹 체감, 게임오버 통계 같은 핵심 구현에 들어간다.

---

## Success Criteria

- 데모 기본 모드를 확정한다: **아레나 MVP** 또는 **기존 서바이벌 런**.
- 서바이벌 런 길이를 유지 기준으로 정한다: **10~20분 압축형** 또는 **30분 정식형**.
- 데모 인벤토리를 런 한정 장비 빌드 시스템으로 노출할 범위를 확정한다.
- 데모 무기 풀 후보를 10~15종으로 줄인다.
- 보스/엘리트/특수몹 중 데모에 고유 체감을 줄 대상을 정한다.
- 골드를 웨이브 사이 상자 비용으로 사용하는 기준을 확정한다.
- F5 메인 루프 기준선 테스트 결과를 기록한다.
- 2주차에 바로 구현할 `Demo Core` 작업 3~5개를 확정한다.

## Decisions Needed

### D01 - 데모 기본 모드와 런 구조

| 선택지 | 장점 | 리스크 |
|--------|------|--------|
| 아레나 MVP | 웨이브 단위로 목표가 명확하고 데모 반복이 빠르다. | 기존 30분 밸런스와 다른 축의 QA가 필요하다. |
| 10~20분 압축 서바이벌 | 현재 생존 루프를 짧게 보여 줄 수 있다. | 아레나와 차별되는 데모 핵심이 약할 수 있다. |
| 30분 정식 서바이벌 | 현재 밸런스 구조와 잘 맞는다. | 데모 유저와 QA에 부담이 크다. |

**결정:** 기본 데모는 **아레나 MVP**로 진행한다. 기존 서바이벌은 로비에서 선택 가능한 비교 모드로 유지한다.

### D02 - 인벤토리 노출

| 선택지 | 장점 | 리스크 |
|--------|------|--------|
| 숨김 | 데모 핵심이 명확하다. | 구현된 인벤토리 가치가 드러나지 않는다. |
| 제한 공개 | F6/테스트 빌드에서만 보여 줄 수 있다. | 공개 데모와 테스트 빌드가 갈라진다. |
| 런 한정 핵심 기능 | 상자와 장비 빌드가 아레나 보상을 만든다. | 기존 영구 저장 경로와 분리해야 한다. |

**결정:** 인벤토리는 런 한정 시스템으로 노출한다. 가방, 장비 세트, 상자 아이템은 현재 런에서만 유지하고 영구 장비 세이브에는 쓰지 않는다.

### D03 - 골드 정책

| 선택지 | 장점 | 리스크 |
|--------|------|--------|
| 유지 | 이미 구현된 HUD와 보상을 살린다. | 소비처가 없어 의미가 약하다. |
| 숨김 | 데모 UI가 단순해진다. | 드랍 보상의 일부가 사라진다. |
| 상자 비용 | 자원의 의미가 생기고 웨이브 사이 선택이 생긴다. | 상자 UI와 추첨 테이블이 필요하다. |

**결정:** 골드는 런 중 자원으로 유지하고, 웨이브 클리어 보상 단계에서 런 한정 상자를 여는 비용으로 사용한다.

## Result

완료한 결정:

- 기본 데모 모드는 **아레나 MVP**로 고정하고, 기존 서바이벌은 로비에서 선택 가능한 비교 모드로 유지한다.
- 데모 인벤토리는 런 한정 장비 빌드 시스템으로 노출하고, 영구 장비 세이브와 메타 진행은 데모 범위에서 제외한다.
- 골드는 런 중 자원으로 유지하며, 웨이브 클리어 후 전체/부위 상자 구매 비용으로 사용한다.
- 아레나는 시간 타임라인이 아니라 웨이브 번호를 난이도 축으로 사용하고, 웨이브 시작은 중앙 텔레포터 상호작용으로 처리한다.
- 아레나 몹은 경험치 오브를 드랍하지 않고, 웨이브 클리어 보상으로 빌드를 보강한다.

완료한 검증:

- Roadmap, Wiki, Architecture, Backlog에 런 한정 인벤토리와 골드 상자 보상 규칙을 1차 반영했다.
- 발사체 이동·충돌·풀링 구조는 `Architecture_Projectiles.md`, 플레이어-facing 규칙은 `Projectiles.md`에서 관리하도록 분리했다.
- F5/F6 수동 플레이 기준선은 아직 완료하지 않았으므로 2주차 QA 항목으로 이관한다.

2주차로 넘길 작업:

- F5 아레나 기준선: 로비 → 시작 무기 → 텔레포터 → 1~10웨이브 → 5/10 보스 웨이브 → 클리어.
- F5 서바이벌 비교 기준선과 F6 테스트 아레나 기준선 확인.
- 데모 무기 풀 10~15종 후보 축소와 F6 대표 movement·피해 통계 검증.
- 보스/엘리트/특수몹 중 고유 체감을 줄 1종 우선 구현.
- 게임오버/클리어 통계와 골드 상자 가격·확률·실패 UX 튜닝.

삭제/보류한 작업:

- 영구 장비 세이브, 메타 장비 진행, 캐릭터 선택, 퀵슬롯, 무기 합성·진화는 데모 이후로 보류한다.
- 30분 정식 서바이벌은 데모 기본 모드가 아니라 비교 가능한 보조 모드로 유지한다.

새로 발견한 리스크:

- 골드 상자 보상이 새 시스템으로 커질 수 있으므로 1~10웨이브는 `Common`/`Uncommon`과 단순 부위 상자로 제한한다.
- 아레나 QA 축이 기존 30분 밸런스와 달라졌으므로 F5 아레나 10웨이브 수동 기준선을 먼저 확보해야 한다.
