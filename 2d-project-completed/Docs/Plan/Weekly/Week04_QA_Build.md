## Goal

4주차의 목표는 에디터 안에서 돌아가는 게임을 **배포 가능한 QA 빌드 후보**로 만드는 것이다. 새 기능 추가는 중단하고, Export, 새 폴더 실행, F5/F6 회귀, 데모 루프 치명 버그를 확인한다.

이번 주가 끝나면 공개 데모로 바로 내지는 않더라도, 외부 테스트에 보낼 수 있는 Windows 빌드 후보와 알려진 문제 목록이 있어야 한다.

---

## Success Criteria

- Windows Export Preset이 생성되고 릴리즈 빌드가 실행된다.
- 에디터 밖 새 폴더에서 빌드를 실행할 수 있다.
- F5 메인 루프가 데모 종료 조건까지 진행된다.
- F6 테스트 아레나에서 데모 무기와 주요 몹을 검증한다.
- 치명 버그, 주요 UX 문제, Polish 항목을 분리한 리스트가 있다.
- 공개 데모 전 남은 작업이 `Hotfix`, `Must`, `Deferred`로 분류된다.

---

## In Scope

- Windows Export Preset 생성과 빌드 검증.
- 프로젝트명, 아이콘, 실행 파일명 최소 정리.
- F5 메인 루프 QA.
- F6 테스트 아레나 회귀.
- 저장/설정 파일 초기화와 새 빌드 실행 확인.
- 성능, 몹 상한, 스폰 피크 확인.
- 치명 버그 수정 또는 명확한 차단 목록 작성.
- 공개 데모용 알려진 문제 목록 작성.

---

## Out of Scope

- 새 콘텐츠 추가.
- 새 무기, 새 몹, 새 시스템 추가.
- 밸런스 대규모 재작성.
- 스토어 페이지 최종 제출.
- Steamworks 세부 등록.
- 모바일, macOS, Linux Export.
- 인벤토리 F5 통합 착수.

## Decisions Needed

### D01 - QA 빌드 판정

| 판정 | 기준 |
|------|------|
| 외부 테스트 가능 | 실행, 시작, 패배/클리어, 재시작이 가능하고 진행 불가 버그가 없다. |
| 내부 QA만 가능 | 실행은 되지만 진행 불가, 치명 UX, 성능 문제가 남아 있다. |
| 빌드 보류 | Export 또는 새 폴더 실행이 실패한다. |

**결정:** 주차 종료 시 기록.

### D02 - 버그 우선순위

| 분류 | 의미 |
|------|------|
| Hotfix | 외부 테스트 전에 반드시 고친다. |
| Must | 공개 데모 전에는 고친다. |
| Deferred | 데모 이후로 넘긴다. |

**결정:** 모든 발견 이슈를 세 분류 중 하나로 배치한다.

## Risks

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Export Preset 생성 지연 | 외부 테스트 불가 | Windows만 우선, 다른 플랫폼 제외 |
| 에디터 밖 경로 문제 | 빌드 실행 실패 | 새 폴더 실행을 가장 먼저 확인 |
| 저장 파일이 이전 테스트와 충돌 | QA 결과 왜곡 | 초기화 방법을 문서화 |
| 성능 피크에서 FPS 급락 | 데모 평가 악화 | 몹 상한, 스폰 밀도, 풀 prewarm 우선 확인 |
| 치명 버그가 많음 | 공개 일정 지연 | Hotfix만 처리하고 나머지는 Deferred 분류 |
