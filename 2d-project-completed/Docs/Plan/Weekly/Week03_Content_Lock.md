## Goal

3주차의 목표는 공개 데모에 들어갈 콘텐츠를 고정하는 것이다. 이 주가 끝나면 데모 무기 풀, 몹 구성, 보스/특수몹 범위, 사운드 최소 세트, 밸런스 기준이 더 이상 크게 흔들리지 않아야 한다.

이번 주 이후에는 새 기능을 추가하지 않고, 4주차 QA 빌드에서 발견된 치명 문제만 수정한다. “있으면 좋은 기능”은 `Demo Polish` 또는 EA/1.0 백로그로 넘긴다.

---

## Success Criteria

- 데모 무기 풀이 최종 확정된다.
- 데모에 등장할 몹 종류와 이벤트 시각이 확정된다.
- 보스/엘리트/특수몹의 데모용 역할이 확정된다.
- 기본 BGM/SFX 최소 세트가 연결되거나, 연결할 파일·위치가 명확히 남는다.
- 데모 런 길이에 맞춘 밸런스 테스트 기준이 확정된다.
- 4주차 QA 빌드에서 테스트할 체크리스트가 완성된다.

---

## In Scope

- 데모 무기 풀 최종 선택.
- 데모에서 제외할 무기, 몹, 시스템 표시.
- 보스/특수몹 연출·보상·패턴의 마무리 범위 확정.
- 기본 사운드 연결 또는 최소 연결 목록 완료.
- 0~데모 종료 시점까지 밸런스 1차 튜닝.
- 게임오버/클리어 통계와 피해 통계 표시 점검.
- QA 체크리스트 작성.

---

## Out of Scope

- 새 무기 추가.
- 새 몹 종류 추가.
- 새 성장 시스템 추가.
- 대규모 UI 재배치.
- Export 플랫폼 확대.
- 스토어 페이지 최종 제출.
- 데모 범위를 바꾸는 기획 변경.

## Decisions Needed

### D01 - 데모 콘텐츠 락 기준

| 기준 | 설명 |
|------|------|
| Must | 없으면 데모가 성립하지 않는 콘텐츠. |
| Polish | 일정이 남으면 넣되, QA 빌드를 밀면 제외한다. |
| Cut | 데모에서 제외하고 EA/1.0 후보로 넘긴다. |

**결정:** 주차 종료 시 각 항목을 Must/Polish/Cut으로 분류한다.

### D02 - 밸런스 튜닝 기준

| 선택지 | 설명 |
|--------|------|
| 압축 데모 기준 | 10~20분 안에 초반/중반/후반을 모두 보여 준다. |
| 30분 기준 | 현재 VS형 타임라인을 유지하고 전체 런을 보여 준다. |

**결정:** 1주차 런 길이 결정에 따른다.
