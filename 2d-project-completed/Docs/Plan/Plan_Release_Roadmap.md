# Plan — Release Roadmap

**상태:** 초안
**작성 목적:** 현재 Godot 2D Survival 프로젝트를 공개 데모, 얼리 액세스, 1.0 출시까지 단계별로 관리하기 위한 실행 로드맵입니다.
**관련:** [`../../AGENTS.md`](../../AGENTS.md), [`../../BACKLOG.md`](../../BACKLOG.md), [`Plan_Balance_VS_Curve_Alignment.md`](Plan_Balance_VS_Curve_Alignment.md), [`../Architecture/Architecture_Inventory.md`](../Architecture/Architecture_Inventory.md)

## 1. Overview

현재 프로젝트는 **한 판 생존 루프가 동작하는 기능 알파 후반** 상태로 본다. 로비, 시작 무기 획득, 몹 스폰, 레벨업 3택1, 30분 클리어, 인벤토리, 밸런스 타임라인, 테스트 아레나는 이미 있다.

출시 관점의 핵심 갭은 **데모 스코프 고정, 무기 성장 체감, 보스·특수몹 고유성, 사운드 연결, 프로젝트 표기, Export Preset, QA, 스토어 자산**이다. 1차 목표는 정식 1.0이 아니라 **Windows 공개 데모**로 잡고, 데모 피드백 이후 얼리 액세스와 1.0 범위를 다시 고정한다.

## 2. Release Targets

| 단계 | 목표 | 권장 범위 | 성공 기준 |
|------|------|----------|-----------|
| 공개 데모 | 외부 유저가 핵심 재미를 확인 | Windows, 1개 맵, 1캐릭터, 10~20분 또는 30분 생존 루프 | 처음 실행부터 클리어/패배까지 진행 가능, 치명 버그 없음 |
| 얼리 액세스 | 반복 플레이와 피드백 수집 | 무기·몹·밸런스 확장, 제한적 메타 진행 | 리뷰 가능한 분량의 콘텐츠와 안정적인 패치 루프 |
| 1.0 | 완성형 출시 | 확정된 콘텐츠 세트, 스토어·법무·운영 준비 | 엔딩/승리 루프, 세이브 호환, QA 체크리스트 통과 |

**1차 플랫폼:** Windows PC
**후순위 플랫폼:** Linux, macOS, 모바일, 콘솔
**배포 후보:** Steam, Itch.io

## 3. Scope Policy

### Demo In Scope

- F5 메인 루프 기준 공개 가능한 한 판 플레이.
- 시작 무기 획득, 레벨업 획득, 리롤·버리기, 자동 선택.
- 30분 클리어 또는 데모용 압축 클리어 타임.
- 보스·엘리트·특수몹 중 최소 1~2종의 고유 체감.
- 런 한정 인벤토리와 웨이브 사이 골드 상자 보상.
- 데모 무기 풀 10~15종 우선 폴리싱.
- 기본 BGM/SFX 연결.
- 게임오버·클리어 통계 표시.
- Windows Export Preset과 배포용 프로젝트명·아이콘.
- 스크린샷, 짧은 설명, 긴 설명, 트레일러 초안.

### Demo Out of Scope

- 영구 장비 세이브와 메타 장비 진행.
- 퀵슬롯 4칸.
- 캐릭터 선택.
- 런 간 메타 진행.
- 무기 합성·진화 전체 시스템.
- 31~40분 하드 모드.
- 모든 무기·몹의 고유 아트 완성.
- 텔레메트리와 자동 업데이트.

## 8. Quality Gates

각 마일스톤 종료 전에 최소 1회 확인한다.

| 게이트 | 확인 내용 |
|--------|-----------|
| Godot 파싱 | 프로젝트 열기·실행 시 파싱 에러 없음 |
| F5 메인 루프 | 로비→게임 시작→무기 획득→자동 장착→스폰→레벨업→패배/클리어 |
| F6 테스트 아레나 | 몹 스폰, 무기 Equip, 플레이어 리스폰 |
| 밸런스 | 9분·11분·25분·28분 이벤트 체감 확인 |
| 성능 | 스폰 피크 구간 FPS와 몹 상한 확인 |
| 저장/설정 | 언어·오디오·소나무 설정 저장과 런 한정 인벤토리 초기화 확인 |
| Export | Windows 릴리즈 빌드 실행, 새 폴더에서 재실행 |
| UI | 1280×720 기준 HUD·메뉴·게임오버·무기 획득 깨짐 없음 |

## 9. Build & Store Checklist

### Build

- Godot 버전 고정.
- `export_presets.cfg` 생성.
- 배포용 프로젝트명, 아이콘, 실행 파일명 확정.
- Debug/Release 빌드 차이 확인.
- 저장 파일 위치와 초기화 방법 문서화.
- 이전 테스트 빌드 세이브 호환 여부 결정.

### Store

- 게임명, 짧은 설명, 긴 설명.
- 핵심 특징 3~5개.
- 시스템 요구사항.
- 스크린샷 6장 이상.
- 30~90초 트레일러.
- 태그·카테고리·연령등급 후보.
- 미디어 키트: 로고, 스크린샷, 개발자 소개, 연락처.
