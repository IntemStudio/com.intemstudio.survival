# Wiki

**역할:** 플레이어에게 드러나는 게임 규칙, 콘텐츠 기획, 용어를 정리하는 문서 묶음입니다.  
**범위:** 코드 구조는 `../Architecture/`, 작업 계획은 `../Plan/`, 개발 작업 지도는 `../../AGENTS.md`를 우선합니다.

---

## 작성 규칙

- Wiki에는 **게임이 어떻게 플레이되어야 하는지**를 적습니다.
- 이미 구현된 규칙과 앞으로 정할 기획을 분리합니다.
- 코드 경로는 필요한 경우에만 짧게 남기고, 긴 구현 설명은 쓰지 않습니다.
- 수치가 튜닝 중이면 “현재값”으로 표시하고, 확정 기획처럼 쓰지 않습니다.
- 새 시스템을 추가하기 전에는 해당 Wiki 문서의 `미정 / 후속` 항목을 먼저 확인합니다.

---

## 상태 표기

| 표기 | 의미 |
|------|------|
| 구현됨 | 현재 플레이 가능한 규칙 |
| 부분 구현 | 데이터나 UI는 있으나 체감/기획이 부족한 규칙 |
| 미정 | 기획 결정을 먼저 해야 하는 규칙 |
| 후속 | 데모 이후 또는 별도 Epic으로 넘길 규칙 |
