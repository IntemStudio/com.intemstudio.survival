## 무기 역할

무기는 한 판 빌드의 핵심 선택지다. 데모 기준으로는 무기 수보다 각 무기의 역할 차이를 우선한다.

| 타입 | 기대 역할 | 현재 공격 방식 |
|------|-----------|----------------|
| Ranged | 안정적인 단일 대상 또는 직선 화력 | 탄환 발사 |
| Melee | 가까운 범위 제압, 관통, 넓은 스윙감 | 검기형 근접 관통 발사체 |
| Magic | 궤도, 추적, 속성 피해, 특수 효과 | 마법 탄·궤도 |
| Throwing / Area | 착지 지점 제어, 독, 장판 | 투척체 후 영역 존 |

공격이 날아가고 맞는 방식은 [`Projectiles.md`](Projectiles.md)에서 관리한다.


## 무기 획득 UI

| 기능 | 현재 규칙 |
|------|-----------|
| 후보 수 | 시작 보상과 레벨업 시 3개 후보를 보여 준다. |
| 획득 결과 | 선택한 무기는 즉시 발사되지 않고 런 인벤토리의 `weapon` 아이템으로 들어간다. |
| 자동 배치 | 활성 `weapon`이 비어 있으면 활성 세트에, 활성 `weapon`이 차 있고 비활성 `weapon`이 비어 있으면 비활성 세트에 바로 장착한다. |
| 가방 이동 | 두 `weapon` 슬롯이 모두 차 있으면 가방에 넣는다. 무기 획득 UI에서 가방도 가득 차면 선택한 무기를 월드 장비 오브젝트로 떨어뜨리고 진행한다. |
| 리롤 | 한 판 최대 3회. 보유·버린 무기를 제외하고 다시 뽑는다. 보유는 가방과 두 `weapon` 슬롯을 모두 포함한다. |
| 버리기 | 한 판 최대 3회. 버린 무기는 이후 후보에서 제외한다. |
| 자동 선택 | 켜면 일정 시간 뒤 우선순위에 맞춰 자동 선택한다. |
| 우선순위 | Ranged, Magic, Melee 순서를 플레이 중 조정할 수 있다. |
| 설명 | 후보에 커서를 올리거나 포커스하면 상세 패널에 설명을 표시한다. |

## 자동 공격

- 활성 전투 세트의 `weapon`만 자동 발사한다.
- 가방에 있거나 비활성 세트에 장착된 무기는 자동 공격 on이어도 발사하지 않는다.
- 자동 공격 토글을 끄면 활성 일반 무기 발사 타이머가 멈춘다.
- 활성 궤도 무기는 회전은 유지하되 피해 판정은 막는다.
- 무기 획득 UI의 자동 선택과 자동 공격 토글은 별도 기능이다.

## 현재 성장 규칙

현재 메인 루프는 레벨업 때 **신규 무기 획득**이 중심이다. 획득한 무기는 빈 weapon 슬롯에 자동 배치되거나 가방으로 들어가며, 활성 전투 세트에 장착된 무기만 실제 공격에 사용된다. 이미 가진 무기의 데미지, 공격 속도, 범위, 투사체 수를 올리는 강화 선택지는 아직 부족하다.

데모 전에는 아래 중 하나를 선택해야 한다.

| 선택지 | 장점 | 리스크 |
|--------|------|--------|
| 보유 무기 강화 추가 | 서바이버류 기대감에 가장 잘 맞음 | 밸런스 작업 증가 |
| 무기 수를 줄이고 기본 스탯 강화 | 구현 범위가 작음 | 성장 선택의 재미가 약할 수 있음 |
| 데모는 신규 무기 중심으로 고정 | 일정 안정 | 장르 기대치와 차이가 큼 |

## 데모 무기 풀 정책

- 전체 카탈로그를 모두 노출하지 않는다.
- 우선 10~15종만 데모 풀로 고른다.
- 같은 역할의 무기가 많으면 하나만 남긴다.
- 공용 아이콘을 쓰는 무기는 데모 풀에서 우선순위를 낮춘다.
- 각 타입마다 최소 2~3개 이상의 선택지를 둔다.

## 무기 품질 체크

데모 풀에 들어가는 무기는 아래를 만족해야 한다.

- 이름과 실제 플레이 감각이 맞는다.
- 툴팁이 공격 방식과 핵심 효과를 설명한다.
- 피해 통계에 올바른 이름으로 잡힌다.
- F6 테스트 아레나에서 기본 몹과 더미 상대로 검증된다.
- F6 테스트 아레나에서 벽·나무 충돌 시 반환, 폭발, 장판 생성 규칙이 확인된다.
- 자동 공격 on/off에서 이상 동작이 없다.
- 풀 반환 또는 수명 종료가 안정적이다.

## 미정 / 후속

- 보유 무기 강화 단계 수.
- 무기 합성·진화 도입 여부.
- 등급을 무기 획득 UI에 표시할지 여부.
- 무기별 고유 아이콘과 색상 규칙.
- 리롤·버리기 비용을 골드나 레벨로 바꿀지 여부.
- 무기 획득과 보유 무기 강화 선택지를 장기적으로 어떻게 공존시킬지.
