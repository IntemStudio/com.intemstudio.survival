## 인벤토리의 기획 역할

인벤토리는 한 판 안에서 장비 구성을 만들고, 무기·보조손·방어구·악세사리를 통해 플레이 스타일을 바꾸는 시스템이다.

인벤토리는 런 한정 시스템이다. 가방, 장비 세트, 상자 아이템, 골드는 현재 런에서만 유지하고 클리어·패배·로비 복귀·새 런 시작 시 초기화한다. 영구 세이브는 해금, 설정, 통계 같은 메타 정보만 다루며 장비 상태는 저장하지 않는다.

## 현재 장비 구조

| 항목 | 규칙 |
|------|------|
| 가방 | 8칸. 각 칸에는 장비나 아이템 하나가 들어간다. |
| 장비 세트 | 2세트. 각 세트는 무기, 헬멧, 갑옷, 장갑, 부츠, 보조손, 악세사리 슬롯을 가진다. |
| 전투 세트 | 현재 전투에 쓰는 장비 세트를 고른다. |
| 공유 방어구 | 방어구 5칸은 두 무기 세트가 공유한다. |
| 무기·보조손 | 두 세트의 무기와 보조손 4칸을 한 화면에 동시에 보여 준다. |
| 빈 슬롯 | 비어 있는 칸은 아무 아이템도 없는 상태로 표시한다. |
| 수명 | 모든 인벤토리 상태는 현재 런 안에서만 유지한다. |

## 장착 규칙

- 같은 아이템은 한 위치에만 존재한다.
- 가방 아이템은 우클릭 또는 더블클릭으로 스마트 장착할 수 있다.
- 가방 또는 장착 슬롯의 장비는 인벤토리에서 왼쪽 Shift+좌클릭으로 바닥에 버릴 수 있다.
- 무기와 보조손은 활성 전투 세트에 장착된다.
- 방어구와 악세사리는 공유 방어구 세트에 장착된다.
- 장비 효과는 장착된 슬롯에서만 적용된다. 가방에 있는 장비는 보유 중이어도 스탯, 패시브, 비주얼 효과를 주지 않는다.
- 양손 무기를 장착하면 해당 세트의 보조손은 비워야 한다.
- 가방이 가득 차서 보조손 반환이 불가능하면 양손 장착은 실패해야 한다.

## 장비 획득과 자동 배치

무기 획득 UI와 상자는 아이템을 먼저 현재 런 인벤토리에 획득시킨다. 획득한 아이템은 대상 장착 슬롯이 비어 있으면 자동 장착하고, 비어 있지 않으면 가방에 넣는다.

`weapon`은 아래 순서로 자동 배치한다.

1. 활성 전투 세트의 `weapon` 슬롯이 비어 있으면 그 슬롯에 바로 장착한다.
2. 활성 세트 `weapon`이 차 있고 비활성 세트 `weapon`이 비어 있으면 비활성 세트 `weapon` 슬롯에 바로 장착한다.
3. 두 `weapon` 슬롯이 모두 차 있으면 가방에 넣는다.
4. 무기 획득 UI에서 가방도 가득 차 있으면 선택한 무기는 월드의 장비 오브젝트로 떨어지고 보상 흐름은 종료된다.
5. 떨어진 장비 오브젝트는 플레이어가 다가가 상호작용을 누르면 다시 획득을 시도한다. 장착 슬롯이 차 있고 가방도 가득 차 있으면 획득하지 못하며, 성공했을 때만 오브젝트가 사라진다.

자동 배치는 장착 편의를 위한 규칙일 뿐이며, 실제 공격은 항상 활성 전투 세트의 `weapon`만 수행한다. 예를 들어 세트 1이 활성이고 새 무기가 빈 세트 2 `weapon`에 들어가면, 새 무기는 세트 전환 전까지 발사하지 않는다.

시작 무기 획득은 같은 규칙을 따른다. 새 런의 첫 선택 시 활성 세트 `weapon`이 비어 있으므로 선택한 무기는 바로 활성 세트에 장착되고, 플레이어는 별도 인벤토리 조작 없이 첫 전투를 시작할 수 있다.

무기가 아닌 장비도 같은 원칙을 따른다. `helmet`, `armor`, `gloves`, `boots`, `accessory`는 공유 방어구 슬롯이 비어 있으면 바로 장착하고, 이미 차 있으면 가방에 넣는다. `offhand`는 weapon과 같은 방식으로 활성 세트부터 확인한다. `offhand 1`은 `weapon 1`이 양손 무기가 아닐 때만 장착할 수 있고, `offhand 2`는 `weapon 2`가 양손 무기가 아닐 때만 장착할 수 있다. 활성 세트 `offhand`가 차 있거나 해당 세트 weapon이 양손이라 막혀 있으면, 비활성 세트 `offhand`도 같은 조건으로 확인한다. 둘 다 불가능하면 가방에 넣는다.

## 장비 버리기

인벤토리에서 가방 또는 장착 슬롯의 장비를 왼쪽 Shift+좌클릭하면 해당 장비를 플레이어 앞 월드 장비 오브젝트로 떨어뜨린다. 버린 장비는 일반 드롭과 동일하게 가까이 다가가 상호작용으로 다시 획득할 수 있다.

버리기는 월드 드롭을 만들 수 있는지 먼저 확인한 뒤 슬롯을 비운다. 드롭 생성에 실패하면 장비는 원래 슬롯에 남아야 하며, 장착 중인 활성 세트 장비를 버리면 즉시 전투 적용도 갱신된다.

## 전투 반영 정책

| 모드 | 규칙 |
|------|------|
| 메인 기본 | 무기 획득 UI와 런 한정 장비 빌드를 함께 사용한다. |
| 테스트 아레나 | 장착된 장비 세트의 무기와 장비 스탯을 전투에서 확인할 수 있다. |
| 저장 정책 | 장비 상태와 상자 보상은 런 종료 후 저장하지 않는다. |

## 장비 효과 적용 규칙

모든 장비 아이템은 장착해야만 효과를 낸다. 상자나 보상으로 획득했지만 가방에만 있는 아이템은 툴팁과 보유 상태만 보여 주며 전투 수치에는 참여하지 않는다.

| 위치 | 효과 적용 |
|------|-----------|
| 활성 세트 `weapon` | 공격, 피해 통계, weapon 기반 스탯 계산에 적용 |
| 비활성 세트 `weapon` | 보유·대기 상태. 세트 전환 전까지 공격하지 않음 |
| 활성 세트 `offhand` | 스탯, 패시브 grant, offhand 비주얼 적용 |
| 비활성 세트 `offhand` | 보유·대기 상태. 세트 전환 전까지 효과 없음 |
| 공유 방어구 슬롯 | `helmet`, `armor`, `gloves`, `boots`, `accessory` 스탯과 패시브 적용 |
| 가방 | 효과 없음 |

무기 외 장비도 획득 시 대상 슬롯이 비어 있으면 자동 장착된다. 자동 장착된 장비는 즉시 스탯과 패시브가 반영된다. 대상 슬롯이 차 있거나 offhand가 양손 무기로 막혀 있어 가방에 들어간 장비는 효과를 내지 않으며, 플레이어가 우클릭, 더블클릭, 드래그 등으로 실제 장착 슬롯에 넣어야 스탯과 패시브가 반영된다.

## 장비 스탯

| 스탯 종류 | 기획 의미 |
|-----------|-----------|
| 피해 배율 | 특정 무기 타입 또는 속성 피해를 강화한다. |
| 이동 배율 | 이동 속도를 바꾼다. |
| 체력 | 최대 체력을 올린다. |
| 방패막이 | 들어오는 피해를 먼저 고정 감소시킨다. |
| 방어력 | 방패막이 이후 피해를 비율 감소시킨다. |
| 패시브 부여 | 보조손이나 장비가 궤도, 가속, 대시 투사체 같은 효과를 준다. |

## 런 패시브 (RunPassive)

레벨업·아레나 웨이브 보상 **3택1**에 패시브가 섞여 나온다(최대 6슬롯·레벨 스택). 일부는 MAX 시 **진화** 패시브로 바뀐다. **활성 세트 악세서리**와 특정 패시브 조합 시 추가 스탯(시너지)이 붙는다. 런 종료 시 초기화한다. 구현·규칙: [`Architecture_Passives.md`](../Architecture/Architecture_Passives.md).

## 보조손

보조손은 무기와 별도의 보조 장비다. 방패처럼 방어를 주거나, 오브처럼 궤도를 주거나, 대시와 연동되는 패시브를 줄 수 있다.

데모에서 보조손을 보여 준다면 아래 기준을 만족해야 한다.

- 한손 무기와 양손 무기의 차이가 명확해야 한다.
- 보조손 비주얼이 플레이어 주변에서 보인다.
- 장착/해제 후 스탯과 패시브가 즉시 반영된다.
- 양손 무기 장착 시 보조손이 숨겨진다.

## 상자와 장비 획득

상자는 골드를 소비해 런 한정 인벤토리에 장비를 추가하는 주요 획득 경로다. 아레나 웨이브 클리어 후 무기 획득이 끝나면 중앙 주변에 전체/부위 상자가 배치되고, 상자 위에는 가격이 표시된다. 플레이어가 상자와 상호작용하면 구매 UI에서 부위, 등급 확률, 보유 골드, 가격, 실패 사유를 확인하고 구매할 수 있다.

| 상자 | 결과 풀 |
|------|---------|
| 전체 상자 | 모든 슬롯 아이템 |
| 무기 상자 | `weapon` |
| 투구 상자 | `helmet` |
| 갑옷 상자 | `armor` |
| 장갑 상자 | `gloves` |
| 신발 상자 | `boots` |
| 보조손 상자 | `offhand` |
| 장신구 상자 | `accessory` |

부위 상자는 해당 슬롯만 나오는 하드 필터다. `offhand` 상자에서는 반드시 `offhand` 아이템만 나온다. 전체 상자는 모든 슬롯을 포함하지만 특정 부위를 보장하지 않는다.

상자에서 얻은 아이템은 획득 상태일 뿐이며, 대상 장착 슬롯이 비어 있으면 자동 장착되고 이미 차 있으면 가방에 들어간다. 해당 장비의 효과는 실제 장착 슬롯에 들어간 뒤부터 적용된다.

상자 구매는 골드 확인, 후보 추첨, 배치 가능성 확인을 통과한 뒤에만 골드를 차감한다. 후보가 없거나 가방이 가득 차서 배치할 수 없으면 구매는 실패하고 골드는 줄어들지 않는다. 다음 웨이브 텔레포터를 시작하면 남은 상자는 사라진다.

### 등급

장비 등급은 `Common`, `Uncommon`, `Rare`, `Epic`, `Legendary`를 기본 단계로 둔다. 아레나 MVP 1~10웨이브 상자는 `Common`과 `Uncommon`만 지급하며, `Rare` 이상은 11웨이브 이후 확장 구간이나 별도 보스 보상에서 다룬다.

| 웨이브 | Common | Uncommon | Rare 이상 |
|--------|--------|----------|-----------|
| 1~2 | 90% | 10% | 0% |
| 3~4 | 80% | 20% | 0% |
| 5 | 70% | 30% | 0% |
| 6~7 | 60% | 40% | 0% |
| 8~9 | 50% | 50% | 0% |
| 10 | 40% | 60% | 0% |

### 추첨 순서

1. 상자 타입으로 슬롯 후보를 제한한다.
2. 현재 아레나 웨이브로 등급을 굴린다.
3. 슬롯, 등급, 해금 조건을 만족하는 아이템 후보를 만든다.
4. 같은 런 안에서 이미 보유한 `item_id`는 제외한다.
5. 후보가 없으면 같은 슬롯의 한 단계 낮은 등급으로 재시도한다.
6. 그래도 후보가 없거나 인벤토리에 배치할 수 없으면 구매를 취소하고 골드를 차감하지 않는다.

## 데모 노출 정책

데모에서는 인벤토리를 영구 장비 세이브가 아니라 아레나 런 안의 선택지로 노출한다. 골드는 상자 비용이 되고, 상자는 웨이브 사이 빌드 보강 수단이 된다.

## 미정 / 후속

- 퀵슬롯 4칸 도입 여부.
- 상자 가격과 등급 확률의 플레이테스트 튜닝.
- Rare 이상 등급 해금 시점과 아이콘 프레임.
- 제작, 강화, 분해 같은 장기 장비 확장.
- 소비 아이템과 포션.
- 영구 해금과 런 한정 인벤토리의 경계.
