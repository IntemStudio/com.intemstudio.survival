## 인벤토리의 기획 역할

인벤토리는 한 판 안에서 장비 구성을 만들고, 무기·보조손·방어구·악세사리를 통해 플레이 스타일을 바꾸는 시스템이다.

현재 구조는 많이 구현되어 있지만, F5 메인 서바이버 루프에서는 기본적으로 전투 반영이 꺼져 있다. 데모에서는 인벤토리를 핵심 기능으로 내세울지, 테스트/후속 기능으로 둘지 먼저 결정해야 한다.

## 현재 장비 구조

| 항목 | 규칙 |
|------|------|
| 가방 | 8칸. 각 칸에는 장비나 아이템 하나가 들어간다. |
| 장비 세트 | 2세트. 각 세트는 무기, 헬멧, 갑옷, 장갑, 부츠, 보조손, 악세사리 슬롯을 가진다. |
| 전투 세트 | 현재 전투에 쓰는 장비 세트를 고른다. |
| 공유 방어구 | 방어구 5칸은 두 무기 세트가 공유한다. |
| 무기·보조손 | 두 세트의 무기와 보조손 4칸을 한 화면에 동시에 보여 준다. |
| 빈 슬롯 | 비어 있는 칸은 아무 아이템도 없는 상태로 표시한다. |

## 장착 규칙

- 같은 아이템은 한 위치에만 존재한다.
- 가방 아이템은 우클릭 또는 더블클릭으로 스마트 장착할 수 있다.
- 무기와 보조손은 활성 전투 세트에 장착된다.
- 방어구와 악세사리는 공유 방어구 세트에 장착된다.
- 양손 무기를 장착하면 해당 세트의 보조손은 비워야 한다.
- 가방이 가득 차서 보조손 반환이 불가능하면 양손 장착은 실패해야 한다.

## 전투 반영 정책

| 모드 | 규칙 |
|------|------|
| 메인 기본 | 서바이버식 레벨업 무기 선택이 중심이며, 장비 전투 반영은 기본 핵심 기능이 아니다. |
| 테스트 아레나 | 활성 장비 세트의 무기와 장비 스탯을 전투에서 확인할 수 있다. |
| 메인 후속 후보 | 인벤토리 무기와 레벨업 무기 선택의 우선순위를 먼저 정해야 한다. |

## 장비 스탯

| 스탯 종류 | 기획 의미 |
|-----------|-----------|
| 피해 배율 | 특정 무기 타입 또는 속성 피해를 강화한다. |
| 이동 배율 | 이동 속도를 바꾼다. |
| 체력 | 최대 체력을 올린다. |
| 방패막이 | 들어오는 피해를 먼저 고정 감소시킨다. |
| 방어력 | 방패막이 이후 피해를 비율 감소시킨다. |
| 패시브 부여 | 보조손이나 장비가 궤도, 가속, 대시 투사체 같은 효과를 준다. |

## 보조손

보조손은 무기와 별도의 보조 장비다. 방패처럼 방어를 주거나, 오브처럼 궤도를 주거나, 대시와 연동되는 패시브를 줄 수 있다.

데모에서 보조손을 보여 준다면 아래 기준을 만족해야 한다.

- 한손 무기와 양손 무기의 차이가 명확해야 한다.
- 보조손 비주얼이 플레이어 주변에서 보인다.
- 장착/해제 후 스탯과 패시브가 즉시 반영된다.
- 양손 무기 장착 시 보조손이 숨겨진다.

## 데모 노출 정책

데모에서는 아래 중 하나를 선택한다.

| 선택지 | 설명 |
|--------|------|
| 숨김 | I 인벤토리를 막고 서바이버 레벨업 루프에 집중한다. |
| 제한 공개 | F6 또는 별도 테스트 빌드에서만 공개한다. |
| 핵심 기능 | 메인 데모에서 장비 전투 반영을 켜고 장비 빌드를 핵심으로 삼는다. |

현재 추천은 **숨김 또는 제한 공개**다. 무기 성장, 보스, 사운드, Export 준비가 데모 필수에 더 가깝다.

## 미정 / 후속

- 메인에서 인벤토리 무기와 레벨업 무기 선택을 어떻게 공존시킬지.
- 퀵슬롯 4칸 도입 여부.
- 장비 드랍, 상점, 제작, 강화.
- 장비 등급과 아이콘 프레임.
- 소비 아이템과 포션.
- 세이브 버전 마이그레이션.
