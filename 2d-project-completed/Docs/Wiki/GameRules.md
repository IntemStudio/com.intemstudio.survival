## 핵심 판 구조

현재 게임은 로비에서 **서바이벌**과 **아레나** 중 하나를 선택해 같은 전투 씬을 다른 규칙으로 시작한다.

1. 로비에서 게임 시작.
2. 서바이벌 또는 아레나 모드를 선택한다.
3. 시작 무기 획득 UI가 열리고 게임은 일시정지된다.
4. 무기를 고르면 런 인벤토리의 빈 `weapon` 슬롯에 자동 배치되고 선택한 모드의 스폰 규칙이 시작된다.
5. 서바이벌에서는 몹을 처치해 경험치 오브와 골드를 얻는다.
6. 레벨업 시 무기 획득 UI에서 3개 후보 중 하나를 고른다.
7. 체력이 0이 되면 패배 게임오버가 열린다.

### 서바이벌 모드

1. 밸런스 표와 타임라인이 시간에 따라 몹 밀도, 몹 종류, 이벤트를 강화한다.
2. 표 기준 30분에 도달하면 스테이지 클리어가 열린다.

### 아레나 모드

1. 시간 강화 대신 웨이브 번호가 난이도 축이 된다.
2. MVP 기준은 1~10웨이브이며, 5/10웨이브는 보스 웨이브다.
3. 시작 무기 획득 후 맵 중앙 텔레포터에서 `E`로 상호작용해야 1웨이브 스폰이 시작된다.
4. 텔레포터 상호작용 직후, 해당 웨이브에 **웨이브 modifier**가 있으면 이름·짧은 설명을 보여 준 뒤 스폰을 시작한다. (구현 전 — [아레나 웨이브 Modifier](#아레나-웨이브-modifier))
5. 아레나 몹은 경험치 오브를 드랍하지 않는다. 성장은 웨이브 클리어 보상(무기·상자)에만 의존한다.
6. 웨이브의 예정 스폰을 모두보내고 필드 몹을 모두 처치하면 웨이브 클리어 보상 단계가 열린다.
7. 보상 단계에서는 무기 획득과 골드 상자 구매로 런 빌드를 보강한다. 가방·장착 슬롯이 차 있으면 **버리기·스왑·정리** 판단이 필요하다.
8. 보상·인벤토리 정리가 끝나면 텔레포터가 다시 활성화된다.
9. 텔레포터에서 `E`로 다시 상호작용해야 다음 웨이브 스폰이 시작된다.
10. 10웨이브를 클리어하면 스테이지 클리어가 열린다.

**서바이벌과의 차이:** 아레나만 웨이브 modifier·웨이브 사이 상자·텔레포터 시작을 사용한다. 서바이벌 타임라인에는 modifier를 넣지 않는다.

## 아레나 MVP 웨이브

아레나 MVP는 100웨이브 구조를 바로 만들지 않고, 1~10웨이브로 흐름을 검증한다. 이후 30/50/70/100웨이브 구간 확장은 후속 밸런스 작업으로 둔다.

| 웨이브 | 구성 의도 | 주요 몹 |
|--------|-----------|---------|
| 1~4 | 기본 몹 적응, 이동과 자동 공격 확인 | Basic, Fast, Ranged 소량 |
| 5 | 첫 보스 관문 | Basic + Boss |
| 6~8 | 밀도 상승과 원거리/엘리트 섞기 | Basic, Fast, Ranged, Elite |
| 9 | 보스 전 압박 | Fast, Ranged, Special A |
| 10 | MVP 최종 보스 웨이브 | Ranged, Elite, Boss |

**웨이브 modifier 배치(MVP-lite, 기획 확정 · 구현 3주차):**

| 웨이브 | Modifier | 의도 |
|--------|----------|------|
| **6** | 국소 위험 지대 | 중반 밀도와 겹쳐 이동·대시 회피 검증 |
| **9** | 회복 감소 | 10웨이브 보스 전 압박, 상자·픽업으로 보완 가능 수준 |

- 1~10 중 **2개 웨이브**만 적용. 승리 조건은 동일(10웨이브 클리어).
- 3주차 구현 전 F5에서 체감이 과하면 웨이브 9를 **엘리트 강화**로 교체 가능.
- 상세 규칙: [아레나 웨이브 Modifier](#아레나-웨이브-modifier), [`GDD.md`](../Design/GDD.md).

웨이브 보상은 무기 획득과 골드 상자 구매를 중심으로 한다. 아레나 몹 처치는 경험치 오브를 만들지 않으며, 무기 획득과 상자 구매는 웨이브 클리어 시점에만 진행한다. 웨이브 종료는 “예정 스폰 없음 + 살아 있는 몹 0”을 기준으로 한다. 첫 웨이브와 이후 모든 다음 웨이브는 자동 시작하지 않고 중앙 텔레포터 상호작용을 기다린다.

무기를 획득하면 활성 전투 세트의 `weapon` 슬롯이 비어 있는지 먼저 확인한다. 활성 `weapon`이 비어 있으면 바로 장착되어 즉시 사용할 수 있고, 활성 `weapon`이 차 있지만 비활성 세트 `weapon`이 비어 있으면 비활성 슬롯에 장착되어 대기한다. 두 `weapon` 슬롯이 모두 차 있으면 가방으로 들어가며, 실제 공격은 항상 활성 전투 세트의 `weapon`만 수행한다.

## 아레나 웨이브 Modifier

**상태:** 미정 (GDD 확정, 코드 미구현)  
**적용:** 아레나 전용. 상시 룰이 아니라 **해당 웨이브가 진행되는 동안만** 적용한다. 웨이브 클리어·패배·다음 텔레포터 대기 시 해제한다.

| Modifier | 규칙 의도 | 제약 |
|----------|-----------|------|
| **국소 위험 지대** | 맵 일부에 지속 피해 또는 디버프 | 전장 전체 위험 금지. 이동·대시로 회피 가능해야 함 |
| **제한 안전 구역** | 작은 구역에서만 회복·대기 허용 | 상시 안전지대 생존 게임이 되면 안 됨 |
| **회복 감소** | 체력 픽업·자연 회복 효율 하락 | 즉사 트랩 금지. 상자·장비로 보완 가능한 수준 |
| **엘리트 강화** | 해당 웨이브 엘리트 스폰·스탯 상승 | 기존 elite/special/boss 역할과 중복 최소화 |

**MVP-lite 범위:** 위 4종 중 **1~2종만** 데모에 넣는다. 복합 modifier·서바이벌 적용·Simulacrum식 독립 모드는 후속.

**UI·연출(미정):**

- 웨이브 시작 전: modifier 이름, 한 줄 설명, 아이콘.
- 위험/안전 구역: 색·테두리·짧은 경고 SFX로 시인성 확보.
- modifier는 **별도 승리 조건이 아니다.**

## 아레나 상자 보상

상자는 골드를 소비해 현재 런 인벤토리에 장비를 추가하는 선택형 보상이다. 웨이브 클리어 후 무기 획득이 끝나면 중앙 주변에 전체/부위 상자가 배치되고, 상자 위 가격 표시와 구매 UI로 보유 골드, 가격, 부위, 등급 확률을 확인한다. 상자와 상자에서 얻은 아이템은 모두 런 한정이며, 클리어·패배·로비 복귀·새 런 시작 시 저장하지 않고 초기화한다.

상자 아이템은 대상 장착 슬롯이 비어 있으면 자동 장착되고, 이미 차 있거나 offhand가 양손 무기로 막혀 있으면 가방에 들어간다. 장비 효과는 실제 장착 슬롯에 들어간 뒤부터 스탯과 패시브가 적용된다.

| 상자 | 결과 풀 |
|------|---------|
| 전체 상자 | 모든 슬롯 아이템 |
| 무기 상자 | `weapon` |
| 투구 상자 | `helmet` |
| 갑옷 상자 | `armor` |
| 장갑 상자 | `gloves` |
| 신발 상자 | `boots` |
| 보조손 상자 | `offhand` |
| 장신구 상자 | `accessory` |

부위가 적힌 상자는 해당 슬롯만 나오는 하드 필터다. 예를 들어 보조손 상자에서는 반드시 `offhand` 아이템만 나온다. 전체 상자는 모든 슬롯을 포함하지만 원하는 부위를 보장하지 않는다.

상자는 후보 없음, 골드 부족, 가방 가득 참 같은 실패 사유를 표시하며 실패 시 골드를 차감하지 않는다. 다음 웨이브 텔레포터를 시작하면 구매하지 않은 상자는 사라진다.

아레나 MVP 1~10웨이브에서는 상자 등급을 `Common`과 `Uncommon`으로 제한한다. `Rare` 이상은 11웨이브 이후 확장 구간, 하드 모드, 별도 보스 보상 같은 후속 범위에서 해금한다.

| 웨이브 | Common | Uncommon | Rare 이상 |
|--------|--------|----------|-----------|
| 1~2 | 90% | 10% | 0% |
| 3~4 | 80% | 20% | 0% |
| 5 | 70% | 30% | 0% |
| 6~7 | 60% | 40% | 0% |
| 8~9 | 50% | 50% | 0% |
| 10 | 40% | 60% | 0% |

## 현재 승리 / 패배

| 상태 | 규칙 |
|------|------|
| 패배 | 플레이어 체력이 0이 되면 게임오버. 재시작은 씬 리로드로 처리한다. |
| 서바이벌 승리 | 밸런스 표 기준 30분에 도달하면 클리어. 필드 몹은 클리어 전용 사망 처리된다. |
| 아레나 승리 | 10웨이브 MVP를 모두 클리어하면 클리어. 웨이브 modifier는 난도 강화일 뿐 승리 조건을 바꾸지 않는다. 이후 30/50/70/100웨이브 확장은 후속 범위다. |
| 클리어 처치 | 클리어로 죽은 몹은 경험치, 골드, 자석, 체력 픽업을 드랍하지 않는다. |
| 클리어 후 입력 | 패배와 동일하게 무기 획득, 일시정지 재개, 자동 공격 입력을 막는다. |

## 데모 기준

공개 데모는 아래 범위를 1차 기준으로 삼는다.

- Windows PC 우선.
- 1개 맵, 1개 캐릭터.
- 기본 데모는 아레나 MVP를 우선한다.
- 기존 서바이벌은 로비에서 선택 가능한 비교 모드로 유지한다.
- 인벤토리는 런 한정 장비 빌드 시스템으로 다루며 영구 장비 세이브와 분리한다. 아레나에서는 웨이브 사이 **장비·가방 정리 판단**이 더 자주 일어나야 한다.
- 아레나 **웨이브 modifier**는 MVP-lite로 1~2종·소수 웨이브만 검증한다. 서바이벌에는 넣지 않는다.
- 무기 수보다 데모 무기 풀의 체감 차이를 우선한다.
- 보스, 엘리트, 특수몹은 최소 1~2종만이라도 고유 체감을 만든다.
- 아레나 변이 규칙은 튜토리얼 없이도 읽혀야 한다(위험/안전 구역 시인성).

## 입력과 메뉴

| 입력 | 현재 규칙 |
|------|-----------|
| 로비 선택 | 서바이벌 시작 또는 아레나 시작. 둘 다 `survivors_game.tscn`으로 들어가지만 런 규칙이 다르다. |
| 이동 | WASD. |
| 공격 | 마우스 왼쪽 버튼. 수동 공격 유지 시 조준 방향으로 발사한다. |
| 상호작용 | E. 장비 줍기와 아레나 텔레포터 시작에 사용한다. |
| 대시 | Space. 현재 대시 중 무적은 없다. |
| 자동 타겟 | F. 자동 타겟 표시와 무기 조준 대상을 켜고 끈다. |
| 자동 공격 | G. 활성 일반 무기의 자동 발사를 켜고 끈다. 사거리 안에 몹이 있을 때만 발사한다. |
| 일시정지 | Esc. 무기 획득, 게임오버, 클리어 중에는 일시정지 메뉴를 열지 않는다. |
| 인벤토리 | I. 현재 런에서 얻은 장비와 상자 보상을 관리한다. |
| 장비 버리기 | 인벤토리에서 왼쪽 Shift+좌클릭. 장착 슬롯 또는 가방 장비를 플레이어 앞 월드 장비 오브젝트로 떨어뜨린다. |
| 전투 세트 스왑 | Tab 또는 닫힌 RMB. 인벤토리 열림 상태에서는 전역 스왑 금지. |

## 맵과 화면

- 메인 맵은 테스트 아레나보다 큰 3배 영역을 사용한다.
- 테스트 아레나는 1배 영역을 사용한다.
- 플레이어 근처 금지 반경을 두고 몹을 스폰한다.
- UI는 FHD 기준으로 배치하고 `UiViewportLayout`으로 실제 화면에 맞춘다.
- 현재 창 기준은 1280x720, 창 크기 변경은 꺼져 있다.

## 확정 전제

- 한 판의 핵심 재미는 “자동 공격 빌드가 강해지는 생존 루프”다.
- **아레나**는 같은 뼈대 위에 **웨이브 변이·인벤토리 판단**으로 전장 압박을 강화한다 ([`GDD.md`](../Design/GDD.md)).
- 데모에서는 시스템 수보다 명확한 성장, 읽히는 적·변이 규칙, 안정적인 F5 플레이를 우선한다.
- 기획 의도 변경은 `GDD.md` → 이 문서 → Plan/BACKLOG 순으로 맞춘다.

## 미정 / 후속

- **아레나 웨이브 modifier:** 웨이브 6(위험 지대)·9(회복 감소) 수치·VFX/SFX·UI (구현 3주차).
- **modifier UI:** 텔레포터 시작 직후 안내 패널·HUD 표기 방식.
- 아레나 확장: 30/50/70/100웨이브, modifier 복합·추가 종류, Rare+ 상자.
- Simulacrum식 **독립 모드** — 데모 범위 밖. 현재는 아레나 웨이브 modifier로만 일부 흡수.
- 서바이벌 런 길이: 10~20분 압축형 vs 30분 정식형.
- 승리 조건 확장: 보스 처치 승리, 맵별 목표.
- 난이도 프리셋: Easy, Normal, Hard.
- 31~40분 이후 하드 모드.
- 캐릭터 선택과 시작 스탯 프리셋.
