## 몹 설계 원칙

- 몹은 시간 경과에 따라 플레이어 빌드의 준비도를 검사한다.
- 초반은 접근형 기본 몹으로 이동과 무기 이해를 돕는다.
- 중반부터 원거리 몹, 엘리트, 특수몹으로 위치 압박을 만든다.
- 후반은 보스와 밀도 이벤트로 빌드 완성도를 확인한다.
- 데모에서는 많은 변종보다 명확히 구분되는 적 3~5종을 우선한다.

## 현재 몹 종류

| 몹 | 현재 역할 | 상태 |
|----|-----------|------|
| basic | 기본 접근형 몹 | 구현됨 |
| fast | 빠른 접근형 압박. 직선 추격 + 점프 추격 기술(쿨·착지 burst). 이동 시 하늘색 꼬리·먼지 | 구현됨 |
| ranged | 거리를 두고 투사체 발사 | 구현됨 |
| elite | 스탯과 비주얼이 강한 변종 | 부분 구현 |
| special_a | 사망 후 지연(기본 3s) → 범위 링 예고 → 주변 폭발(플레이어 범위 피해) | 부분 구현 |
| special_b | 트리거 거리 내 경로 예고 후 돌진·종료 범위 피해·저체력 자폭(사망 burst export 유지) | 부분 구현 |
| boss | 25분 보스 후보 | 부분 구현 |
| dummy | 테스트 아레나 전용 허수아비 | 구현됨 |

## 등장 리듬

현재 밸런스는 VS형 30분 런을 기준으로 한다.

| 시각 | 기대 체감 |
|------|-----------|
| 0~4분 | 기본 몹 중심. 이동, 공격, 경험치 수집 학습. |
| 5분 전후 | 빠른 몹 비중 증가. 초반 빌드 검증. |
| 8분 이후 | 원거리 몹 등장. 정지 플레이를 방해. |
| 9분 | 엘리트 단발 이벤트. |
| 11분 | 1차 밀도 피크. 준비 안 된 빌드가 흔들리는 구간. |
| 16~20분 | 호흡 구간. 밀도는 낮추고 체력 높은 적 체감. |
| 25분 | 보스 강제 스폰. |
| 28분 | 후반 밀도 이벤트. |
| 30분 | 클리어. 필드 몹 전멸, 드랍 없음. |

## 빠른 몹 (fast)

- 기본 몹보다 높은 이동 속도(`speed_min`/`speed_max`)로 접근 압박을 만든다.
- **추격 방식**은 직선(`chase_mode=0`)이다. 포위와 달리 플레이어 쪽으로 곧바로 접근한다.
- **점프 추격 기술**(`jump_chase_enabled`)이 켜져 있다. standoff 밖·발동 거리 안·쿨다운이 돌아오면 머리 위 `!` 예고 후 짧게 호핑해 앞으로 이동하고, 착지 위치에서 반경 피해(burst)를 준다. 평소에는 직선 추격을 유지하며, 기술은 그 **도중**에 주기적으로 섞인다.
- burst 반경·피해만 0으로 두면 호핑 이동만 하고 피해는 없다(이동과 해결 효과 분리).
- 이동 중에는 발밑 뒤로 **하늘색 Line2D 꼬리**와 **짧은 먼지 파티클**이 보여 빠르다는 것을 한눈에 구분한다.
- 멈추거나 둔화되면 트레일이 꺼진다. 풀 반환·사망 시 트레일·기술 상태·예고 마크도 즉시 사라진다.
- F6에서 **추격 기술** 섹션으로 활성·발동 거리·예고·점프 거리·쿨·착지 burst를 튜닝할 수 있다. **추격 방식** 드롭다운과는 별개다.
- 구현: `mob_fast.tscn`(`jump_chase_enabled=true`), `MobChaseSkillJump` + `MobChaseSkillEffectLandingBurst`, 공통 AI는 `mob.gd`. 이동 FX는 자식 `SpeedTrail`(`mob_speed_trail.tscn`).

## 근접 몹

- 플레이어에게 접근한다.
- 접촉형 몹만 범위 주기 공격과 최초 충돌 피해를 가진다.
- 원거리 몹은 접촉 피해가 없다.
- 몹 분리 로직으로 서로 겹치지 않게 밀어낸다.

## 원거리 몹

- 공격 거리 밖에서는 추적하고, 안에서는 멈춰 공격한다.
- 발사 전 머리 위에 예고 마크를 띄운다.
- 예고 시간이 끝나면 플레이어 방향으로 투사체를 발사한다.
- 투사체는 환경 또는 플레이어에 닿으면 사라진다.

## 엘리트 / 특수몹 / 보스 정책

현재 엘리트, 특수몹, 보스는 이름과 수치에 비해 고유 gameplay가 부족하다. 데모 전에는 아래처럼 최소 역할을 정한다.

| 분류 | 데모 최소 목표 |
|------|----------------|
| elite | 일반몹보다 크고 단단하며, 등장 알림 또는 보상 차별화가 있다. |
| special_a | 사망 위치에서 `death_burst_delay`(기본 3s) 후 반경 피해. 지연 동안 링이 커지며 예고. F6에서 범위·피해·지연 튜닝. 이동·공격 패턴 차별은 후속. |
| special_b | `charge_trigger_distance` 안에서 **레인·`!` 예고**(`charge_lane_display_duration`, 기본 1s) 후 돌진, 종료 반경 피해. HP 임계 **자폭**. F6 GUI는 **돌진 거리** 튜닝(사망 폭발 스핀 숨김). |
| boss | 25분에 등장하며, 고유 공격과 보상 또는 클리어 압박을 가진다. |

## 보상 규칙

- 몹 종류와 현재 진행 구간에 따라 XP와 골드 보상이 달라진다.
- boss, elite, special은 기본 XP가 더 높다.
- 더미는 테스트용이라 보상이 없다.
- 엘리트/보스 전용 드랍 테이블과 오브 비주얼 차등은 후속이다.

## 미정 / 후속

- 몹 종류 값을 AI, 보상, UI 분기에 적극 사용할지 여부.
- 위협도 지표를 피해, 스폰, 압박도 중 어디에 연결할지.
- 보스 페이즈와 패턴 수.
- 엘리트 등장 연출과 처치 보상.
- 특수몹 A/B의 최종 정체성.
- 동시 생존 몹 상한을 HUD 또는 디버그 UI에 표시할지 여부.
