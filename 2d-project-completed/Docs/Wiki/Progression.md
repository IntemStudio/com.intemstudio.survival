## 성장 구조

서바이벌 성장은 “몹 처치 → 경험치 수집 → 레벨업 → 무기 선택” 흐름이 중심이다. 아레나는 처치 경험치 대신 웨이브 클리어 시 무기 선택을 제공한다.

1. 서바이벌에서 몹을 처치하면 경험치 오브와 골드가 떨어진다.
2. 플레이어가 오브를 수집하면 경험치를 얻는다.
3. 경험치가 목표치를 넘으면 레벨업한다.
4. 레벨업 선택 UI에서 무기 후보를 고른다.
5. 선택한 무기가 빌드에 추가된다.

## 경험치

| 항목 | 규칙 |
|------|------|
| 드랍 | 몹 종류와 현재 진행 구간에 따라 경험치량이 정해진다. |
| 수집 | 경험치 오브가 플레이어의 수집 범위에 들어오면 자석처럼 빨려 온다. |
| 레벨업 | 누적 경험치가 기준치를 넘으면 레벨업하고 무기 선택 UI가 열린다. |
| 아레나 | 몹 처치 경험치는 0이며, 경험치 오브 대신 웨이브 종료 시 무기 선택 UI가 열린다. |
| 클리어 사망 | 30분 클리어로 사라지는 몹은 XP를 드랍하지 않는다. |

## 골드

| 항목 | 규칙 |
|------|------|
| 드랍 | 몹 처치 보상 XP를 기준으로 골드가 계산된다. |
| 수집 | 골드 코인은 경험치 오브처럼 자석 수집된다. |
| 표시 | HUD에 현재 골드가 표시된다. |
| 소비 | 현재 소비처는 없다. 데모에서는 숨김 또는 최소 소비처 결정이 필요하다. |

## 픽업

| 픽업 | 현재 규칙 |
|------|-----------|
| 경험치 오브 | 주요 성장 자원. 풀링되어 재사용된다. |
| 골드 코인 | 런 중 수집 자원. 풀링되어 재사용된다. |
| 자석 | 낮은 확률로 드랍. 필드의 경험치와 골드를 끌어당긴다. |
| 체력 | 낮은 확률로 드랍. 플레이어 체력을 회복한다. |

## 한 판 안 성장의 목표

- 초반에는 무기 종류가 늘어나는 재미를 준다.
- 중반에는 선택한 빌드의 강점과 약점이 보인다.
- 후반에는 보스와 밀도 이벤트를 버틸 만큼 성장했는지 확인한다.
- 데모 기준으로는 “선택이 빌드를 바꿨다”는 체감이 가장 중요하다.

## 현재 큰 갭

| 갭 | 영향 |
|----|------|
| 보유 무기 강화 부족 | 레벨업 선택이 신규 무기 수집에 치우친다. |
| 골드 소비처 없음 | HUD에 표시되는 자원의 의미가 약하다. |
| 메타 진행 없음 | 여러 판 반복 동기가 약할 수 있다. |
| 패시브·액세서리 성장 없음 | 빌드 다양성이 무기 종류에만 의존한다. |

## 데모 선택지

데모에서는 아래 중 하나를 선택한다.

| 선택지 | 설명 |
|--------|------|
| 골드 숨김 | 소비처가 없으면 데모 UI에서 골드 노출을 줄인다. |
| 간단 상점 | 런 중 리롤, 회복, 일회성 강화 비용으로 골드를 쓴다. |
| 런 종료 보상 | 클리어/패배 후 획득 골드만 보여 주고 소비는 EA로 넘긴다. |

## 미정 / 후속

- 레벨업 곡선과 필요 경험치 튜닝.
- 엘리트/보스 전용 경험치와 드랍 비주얼.
- 골드 소비처: 상점, 리롤 비용, 메타 업그레이드.
- 런 간 영구 해금.
- 런 종료 리포트 JSON 저장.
- 플레이타임, 처치 수, 무기 선택 통계 수집.
