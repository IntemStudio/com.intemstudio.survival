## Gameplay Gaps

- [ ] **특수 몹 A/B** — HP·속도·색 외의 이동/공격 패턴 정의.
- [ ] **보스** — 페이즈, 고유 스킬, 등장 연출, 처치 보상 차별화.
- [ ] **엘리트** — 전용 기믹, 등장 연출, 보상 차별화.
- [ ] **위협도 지표 활용** — 현재 밸런스 데이터의 threat 값을 피해, 스폰, HUD 중 어디에 연결할지 결정.
- [ ] **몹 종류 값 활용 확대** — AI, 보상, UI 분기에 적극 사용할지 결정.
- [ ] **레이피어 “En Garde”** — 웨이브 시작 시 8초 동안 공격 속도 +20% 버프를 적용하는 버프 시스템 구현.
- [ ] **무기 등급 노출** — 무기 획득 UI, 드롭 가중치, 성장 연동 여부 결정.
- [ ] **승리 조건 확장** — 보스 처치 승리, 웨이브 클리어, 맵별 목표.
- [ ] **캐릭터 선택** — HappyBoo 외 캐릭터와 시작 스탯 프리셋.
- [ ] **메타 진행** — 런 간 영구 해금·통화·업그레이드.

## Balance & Progression

- [ ] **VS D — 몹 HP × 플레이어 레벨** — 강한 빌드일수록 적도 단단해지는 보정 검토.
- [ ] **VS E — 하이퍼 모드** — 25분부터 전역 배율과 “HYPER” 표시 연동.
- [ ] **VS F — 플레이테스트·튜닝** — 키프레임, 밀도 이벤트, 보상 배율 미세 조정.
- [ ] **31~40분 구간 정책** — 30분 클리어 유지 vs 클리어 후 하드 모드 확장.
- [ ] **접촉 피해 미세 튜닝** — 무적 프레임, 변종별 공격 거리·주기, 고속 탄 터널링 보강.
- [ ] **동시 생존 몹 상한 UX** — 스폰 스킵 상태를 HUD 또는 디버그 UI에 표시할지 결정.
- [ ] **난이도 프리셋** — Easy/Normal/Hard용 밸런스 리소스 분리.
- [ ] **레벨업 곡선 튜닝** — 필요 경험치와 보상량 조정.
- [ ] **엘리트/보스 전용 보상** — 전용 드랍 테이블과 오브 비주얼 차등.

## Inventory & Items

- [ ] **퀵슬롯 4칸** — UI, 데이터, 입력 정책을 별도 Epic으로 분리.
- [ ] **런 한정 인벤토리 적용** — 새 런 시작, 클리어, 패배, 로비 복귀 시 가방·장비 세트·골드 초기화.
- [ ] **장비 획득 자동 배치 구현** — 빈 `weapon`/`offhand`/공유 방어구 슬롯은 자동 장착하고, 막히거나 차 있으면 가방으로 배치.
- [ ] **장비 효과 장착 게이트 구현** — 가방 장비와 비활성 weapon/offhand는 스탯·패시브·비주얼·공격 효과에서 제외.
- [ ] **상자 가격·확률 튜닝** — 전체/부위 상자 가격, 1~10웨이브 `Common`/`Uncommon` 확률, 골드 수급을 플레이테스트로 조정.
- [ ] **상자 UI/비주얼 폴리싱** — 상자 아이콘, 등급 프레임, 구매 결과 연출, 골드 부족/가방 가득 참 안내 개선.
- [ ] **Rare 이상 장비 확장** — 11웨이브 이후, 하드 모드, 보스 보상에서 해금할 등급과 아이콘 프레임 정리.
- [ ] **소비 아이템과 포션** — 가방/퀵슬롯/전투 입력 정책 결정.
- [ ] **영구 세이브 경계 정리** — 장비 상태는 저장하지 않고 해금·설정·통계만 저장하도록 기존 저장 경로 조정.

## Ideas

구현 여부와 우선순위는 플레이테스트 후 결정합니다. 기각해도 됩니다.

- 플레이어 주변 링/오프스크린 스폰.
- 보스 처치 시 일시 무적, 전체 경험치, 상자 드롭.
- 무기 획득 리롤·버리기 골드/레벨 비용.
- 5분마다 30초 고밀도 같은 시간 제한 이벤트.
- 소나무 파괴, 불 장판 등 환경 상호작용.
- 플레이어 넉백, 대시 무적.
- 몹 투사체 전용 physics layer 분리.
